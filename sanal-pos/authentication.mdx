Sanal POS API'si Payven'in standart OAuth 2.0 + JWT Bearer akışını kullanır. Token Identity servisinden alınır, Sanal POS endpoint'lerine `Authorization: Bearer <token>` ile gidersiniz.

<Note>
**Genel akış, SDK örnekleri ve token süreleri** için kanonik [Kimlik Doğrulama](/documentation/concepts/authentication) sayfasına bakın. Bu sayfa Sanal POS'a özgü endpoint URL'lerini ve örneklerini içerir.
</Note>

## Endpoint URL'leri

| Ortam | Identity (token alma) | Sanal POS API |
|---|---|---|
| **Production** | `https://identity.payven.com.tr` | `https://vpos.payven.com.tr` |
| **Sandbox** | `https://identity-sandbox.payven.com.tr` | `https://vpos-sandbox.payven.com.tr` |

## Hızlı başlangıç

```bash
# 1. Token al
TOKEN=$(curl -s https://identity.payven.com.tr/api/v1/auth/payven/token \
  -H "Content-Type: application/json" \
  -d '{
    "client_id":     "'"$PAYVEN_CLIENT_ID"'",
    "client_secret": "'"$PAYVEN_CLIENT_SECRET"'"
  }' | jq -r .access_token)

# 2. Sanal POS API'yi çağır
curl https://vpos.payven.com.tr/api/v1/payments \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: pay-2026-05-05-abc123" \
  -d '{
    "amount": 10000,
    "currency": "TRY",
    "card_number": "4111111111111111",
    "expiry_month": 12,
    "expiry_year": 2030,
    "cvv": "123",
    "installment": 1
  }'
```

## Merchant kimliği

Bir token, tenant'a bağlı **tüm merchant'lar adına** işlem yapabilir. Hangi merchant adına işlem yapıldığını request header'larıyla belirtirsiniz:

```http
X-Merchant-Id:          3fa85f64-5717-4562-b3fc-2c963f66afa6
```

Veya kendi sisteminizdeki kimlik ile:

```http
X-External-Merchant-Id: M-IST-001
```

İki header'dan **biri** zorunludur. Her ikisi gönderilirse `X-Merchant-Id` öncelikli olur.

| Header | Tip | Anlam |
|---|---|---|
| `X-Merchant-Id` | UUID | Konsoldan kopyalanır (Merchant.Id) |
| `X-External-Merchant-Id` | Serbest metin | Onboarding sırasında belirlenir (Merchant.ExternalId) |

## Yanıt header'ları

Her response'da Payven aşağıdakileri ekler:

```http
X-Correlation-Id:       9f1c8e76-2a3b-4f12-9c8d-12cb24a8a8a8
X-RateLimit-Limit:      200
X-RateLimit-Remaining:  187
X-RateLimit-Reset:      1746450896
```

| Header | Açıklama |
|---|---|
| `X-Correlation-Id` | Bu isteğin Payven log zincirindeki kimliği. Destek talebi açarken paylaşın. |
| `X-RateLimit-*` | Mevcut kota. [Detay](/documentation/concepts/rate-limiting). |
| `Retry-After` | Yalnız `429` yanıtında — kaç saniye sonra tekrar denemeniz gerektiği. |
| `Idempotent-Replayed` | Yalnız idempotent replay'lerde — yanıt cache'den geldiyse `true`. |

## Hata yanıtları (kimlik doğrulama)

| HTTP | `code` | Anlam | Çözüm |
|---|---|---|---|
| `401` | `invalid_token` | `Authorization` header'ı eksik / geçersiz | Token alın veya refresh edin |
| `401` | `token_expired` | Access token süresi doldu | Refresh akışı |
| `403` | `merchant_inactive` | Merchant pasif statüde | Konsol → Merchants |
| `403` | `merchant_not_found` | Belirtilen merchant bulunamadı | Header'ı doğrulayın |
| `403` | `product_not_licensed` | Sanal POS modülü planınızda etkin değil | Plan yükseltme |
| `429` | `rate_limit_exceeded` | Limit aşıldı | `Retry-After` header'ına uyun |

Tam hata kataloğu için: [Hata Yönetimi](/documentation/concepts/errors).

## Güvenlik kuralları

<Check>**Token cache** — Her API çağrısında token alıp Identity'yi yormayın. Bellek-içi cache + 60sn margin ile auto-refresh kullanın.</Check>
<Check>**Sadece sunucu tarafı** — `client_secret`'ı frontend, mobil veya public repo'ya **asla** koymayın.</Check>
<Check>**HTTPS zorunlu** — HTTP istekleri reddedilir.</Check>
<Check>**Production = ayrı client** — Production ve sandbox için ayrı `client_id`/`client_secret` kullanın.</Check>
<Check>**Loglarda maskele** — `access_token`, `refresh_token`, `client_secret` değerlerini log'lara yazmayın.</Check>
