Sanal POS hata kodları **üç kategoriye** ayrılır:

| Kategori | Kaynak | Örnek değerler |
|---|---|---|
| **Payven domain kodları** | Payven iş kuralı + connector katmanı | `PV_INSUFFICIENT_FUNDS`, `PV_BANK_DECLINED`, `THREEDS_VERIFY_FAILED`, `REFUND_AMOUNT_EXCEEDS_REMAINING` |
| **HTTP canonical kodlar** | Generic 4xx/5xx altyapı | `bad_request`, `unauthorized`, `validation_failed`, `rate_limit_exceeded`, `idempotency_key_in_use` |
| **Provider (banka) kodları** | İşlemi yürüten banka — ISO 8583 spec'i | `00` (başarı), `51` (yetersiz bakiye), `54` (expired card) |

Yanıttaki alan eşleşmesi:

| Yanıt alanı | Kategori |
|---|---|
| `code` (problem+json) | Payven domain veya HTTP canonical |
| `error_code` (PaymentOperationResult.extra) | Payven domain (örn. `PV_BANK_DECLINED`) |
| `provider_error_code` | Banka (örn. `51`) |

## Hata zarfı

Tüm hatalar [RFC 9457 problem+json](/documentation/concepts/errors) zarfında döner:

```json
{
  "type":           "https://docs.payven.com.tr/errors/PV_INSUFFICIENT_FUNDS",
  "title":          "Yetersiz bakiye",
  "status":         422,
  "code":           "PV_INSUFFICIENT_FUNDS",
  "detail":         "Banka kart bakiyesinin yetersiz olduğunu bildirdi.",
  "instance":       "/api/v1/payments/8e3f5c12-...",
  "correlation_id": "9f1c8e76-2a3b-4f12-9c8d-12cb24a8a8a8"
}
```

Banka tarafı reddinde HTTP `200` döndüğü ve hata bilgisinin `PaymentOperationResult` gövdesinde `error_code` + `provider_error_code` alanlarında geldiği özel duruma da dikkat edin — bkz. [Payment Objesi](/sanal-pos/payment-object).

## Konsoldaki canlı katalog

Sürekli güncellenen **konnektör hata kodu kataloğu** konsolda mevcuttur:

→ [Konsol → Konnektör Hata Kodları](https://dashboard.payven.com.tr/connector-error-codes)

Bu ekran her bankanın özel kod açıklamalarını, Payven domain kodu eşlemesini ve önerilen kullanıcı mesajlarını içerir.

## Hata yönetimi pratik kılavuz

| Durum | Yapılacak |
|---|---|
| `validation_failed`, `invalid_*` | İsteği düzeltip yeniden gönderin (retry yok) |
| `unauthorized`, `invalid_token` | Token alın / refresh edin |
| `forbidden`, `merchant_inactive`, `product_not_licensed` | İlgili kuruluş başkanlığıyla / Payven satışla iletişime geçin |
| `rate_limit_exceeded` | `Retry-After` header'ına uyup backoff ile retry edin |
| `idempotency_key_in_use` | Aynı anahtarla farklı body göndermeyi durdurun veya farklı anahtar üretin |
| `PV_BANK_DECLINED` | Müşteriye nazik mesaj gösterip başka kart önerin (`provider_error_code` daha spesifik bilgi) |
| `PV_BANK_TIMEOUT` | Smart Retry zaten devreye girer — istemci tarafında ek bir şey yapmayın |
| `PV_FRAUD_SUSPECTED` | Müşteriye genel "işlem onaylanmadı" mesajı gösterin (spesifik sebep paylaşmayın — fraud signal'i sızdırır) |
| `5xx` (sunucu hatası) | Aynı `Idempotency-Key` ile exponential backoff retry |

Genel hata yönetimi prensibi ve retry kodu örnekleri: [Hata Yönetimi (Konsept)](/documentation/concepts/errors).
