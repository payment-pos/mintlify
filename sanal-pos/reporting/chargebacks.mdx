Chargeback, kart sahibinin bankasına başvurarak işlemi reddetmesi durumunda **bankanın size karşı açtığı geri alma talebidir**. Cevap vermez veya kaybederseniz tutar hesabınızdan çekilir.

<Note>
Şu anda Payven chargeback **kayıt + status takibi** destekler — ham yanıt evrakının
bankaya iletimi konnektör + banka iş süreçleri üzerinden gerçekleştirilir.
Programatik "respond + evidence upload" akışı yol haritasındadır.
</Note>

## Endpoint'ler

```http
GET    /api/v1/chargebacks                  # Liste (sayfalı)
GET    /api/v1/chargebacks/{id}             # Tek chargeback detayı
POST   /api/v1/chargebacks                  # Manuel chargeback kaydı (banka tarafından bildirilen)
PUT    /api/v1/chargebacks/{id}/status      # Durum güncelle
```

Bearer access token gerektirir (OAuth 2.0 Client Credentials).

## Liste

```bash
curl "https://vpos.payven.com.tr/api/v1/chargebacks?status=open&page=1&page_size=20" \
```

## `status` değerleri

| Status | Anlam |
|---|---|
| `open` | Bankaya yanıt verilmesi bekleniyor |
| `responded` | Yanıt gönderildi, banka kararı bekleniyor |
| `won` | Lehinize karara bağlandı, tutar kalır |
| `lost` | Aleyhinize karara bağlandı, tutar düşüldü |
| `withdrawn` | Müşteri talebini geri çekti |

## Manuel chargeback kaydı

Banka chargeback bildirimini Payven dışından (e-posta, banka portalı) aldıysanız sisteme manuel kaydedebilirsiniz:

```bash
curl -X POST https://vpos.payven.com.tr/api/v1/chargebacks \
  -H "Authorization: Bearer $PAYVEN_TOKEN" \
  -H "Idempotency-Key: cb-bank-ref-789" \
  -H "Content-Type: application/json" \
  -d '{
    "transaction_id":     "8e3f5c12-...",
    "amount":             15000,
    "currency":           "TRY",
    "reason_code":        "4837",
    "reason_description": "İşlem yetkisiz",
    "opened_at":          "2026-05-15T10:00:00+00:00",
    "response_deadline":  "2026-05-22T23:59:59+00:00",
    "bank_reference":     "CB-GAR-789"
  }'
```

## Status güncelleme

Bankaya yanıt gönderdikten veya banka kararını öğrendikten sonra:

```bash
curl -X PUT https://vpos.payven.com.tr/api/v1/chargebacks/9f3d2b8e.../status \
  -H "Authorization: Bearer $PAYVEN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status":   "responded",
    "remarks":  "Müşteri ile yazışmalar + kargo teslim fişi banka portalına yüklendi (REF: ABC-123)"
  }'
```

`remarks` alanı operatör notları için kalıcı saklanır — audit/dispute süreci için kritik.

## Reason kodları (Visa/Mastercard)

Sık karşılaşılan kodlar:

| Kod | Açıklama (Visa/MC) |
|---|---|
| `4837` | İşlem yetkisiz (No Cardholder Authorization) |
| `4855` | Mal/hizmet teslim edilmedi |
| `4853` | Mal/hizmet kalitesinde sorun |
| `4863` | Cardholder Does Not Recognize |
| `4870` | Sahtecilik şüphesi (chip/non-chip) |
| `4880` | Yetkisiz tekrar tahsilat |

Tam liste için banka anlaşma dokümanlarınıza veya kart şeması sözlüğüne (Visa Reason Codes / MC Chargeback Reason Codes) bakın.

## Süre kısıtı

Chargeback'lere genellikle **7 gün** içinde yanıt verilmesi beklenir (banka anlaşmanıza göre değişir). Süreyi geçirirseniz banka çoğu durumda otomatik müşteri lehine karar verir.

## Önleme prensipleri

<Check>**3D Secure kullanın** — başarılı 3DS sonrası chargeback sorumluluğu bankaya geçer (özellikle 4837 kategorisi için kritik).</Check>
<Check>**Müşteri iletişim kayıtlarını saklayın** — yazışmalar, sipariş onayı, teslim kanıtı.</Check>
<Check>**Net merchant adı** — kart ekstresinde müşterinin tanıyacağı isim kullanın (`Acme Magaza` yerine `XYZ Bilgi Tek` müşteriyi şüpheye düşürür).</Check>
<Check>**Hızlı iade politikası** — müşteri sizden iade alabilse chargeback açmaz; en pratik chargeback önleme yolu.</Check>
<Check>**Fraud kuralları** — yüksek riskli işlemleri önceden işaretleyin (kurallar konsol → Fraud ekranından yönetilir).</Check>

## Yol haritası

- **Programatik evidence upload** — `POST /chargebacks/{id}/evidence` (PDF/JPG attach)
- **Webhook event'i** — `chargeback.created`, `chargeback.resolved` (şu an public webhook event listesinde yok)
- **Visa/MC reason kodu kataloğu** — konsoldan açıklama görüntüleme
