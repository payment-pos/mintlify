**Konnektör**, Payven'in bir bankaya nasıl bağlandığını tanımlayan tip katmanıdır (örn. *NestPay*, *HalkBankVPOS*). Kendisi credential içermez — bunu [Konnektör Konfigürasyonları](/sanal-pos/backoffice/configurations) yapar.

## Endpoint'ler

```http
GET    /api/v1/connectors                  # Liste (sayfalı)
GET    /api/v1/connectors/{id}             # Tek konnektör (özet)
GET    /api/v1/connectors/{id}/detail      # Tek konnektör (genişletilmiş — yetenekler dahil)
POST   /api/v1/connectors                  # Yeni konnektör tipi
PUT    /api/v1/connectors/{id}             # Güncelleme
DELETE /api/v1/connectors/{id}             # Silme (soft)
POST   /api/v1/connectors/{id}/activate
POST   /api/v1/connectors/{id}/deactivate
GET    /api/v1/connectors/{id}/health      # Sağlık durumu
```

JWT Bearer auth gerektirir.

## Liste

```bash
curl https://vpos.payven.com.tr/api/v1/connectors \
  -H "Authorization: Bearer $PAYVEN_TOKEN"
```

## Detay (yetenek matrisi)

```bash
curl https://vpos.payven.com.tr/api/v1/connectors/abc-.../detail \
  -H "Authorization: Bearer $PAYVEN_TOKEN"
```

Yanıt; konnektörün desteklediği operasyonları (Sale, 3DS, Refund, Void, Capture, Recurring, OrderLink, PointInquiry, OrderHistoryQuery) ve her birinin durum bilgisini taşır.

## Sağlık endpoint'i

```bash
curl https://vpos.payven.com.tr/api/v1/connectors/abc-.../health \
  -H "Authorization: Bearer $PAYVEN_TOKEN"
```

Anlık başarı oranı, ortalama yanıt süresi, son hata zamanı, circuit breaker durumu döner.

## Yeni banka talebi

Listede olmayan bir banka için entegrasyon talebinde bulunmak üzere [destek ekibimize](/resources/support) yazın. Yeni konnektör geliştirme süresi tipik 4-8 hafta arasıdır.

## İlişki

- Bir **konnektör** birden çok **konnektör konfigürasyonu**na sahip olabilir (farklı merchant'lar için ayrı credential setleri)
- Konfigürasyon detayları için: [Konnektör Konfigürasyonları](/sanal-pos/backoffice/configurations)
- Sağlık + circuit breaker: [Circuit Breaker](/sanal-pos/routing/circuit-breaker)
