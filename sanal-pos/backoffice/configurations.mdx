---
title: "Konnektör Konfigürasyonları"
description: "Banka credential'larını yönetin, sağlık durumunu izleyin."
---

Konfigürasyon, bir konnektör tipini **kendi banka anlaşmanız** için ayarladığınız credential setidir (üye işyeri numarası, terminal ID, store key, API key/secret, vb.). Bir konnektörün birden fazla konfigürasyonu olabilir — örn. farklı merchant'lar için ayrı credential'lar.

## Endpoint'ler

```http
GET    /api/v1/connector-configurations              # Liste (sayfalı)
GET    /api/v1/connector-configurations/{id}         # Tek konfigürasyon
POST   /api/v1/connector-configurations              # Yeni
PUT    /api/v1/connector-configurations/{id}         # Güncelleme
DELETE /api/v1/connector-configurations/{id}         # Silme (soft)
POST   /api/v1/connector-configurations/{id}/activate
POST   /api/v1/connector-configurations/{id}/deactivate
GET    /api/v1/connector-configurations/{id}/health  # Sağlık durumu (başarı oranı, latency, circuit breaker)
```

Bearer access token gerektirir (OAuth 2.0 Client Credentials).

## Oluşturma

Her konnektörün kendine özgü credential alanları vardır. NestPay için örnek:

```bash
curl -X POST https://vpos.payven.com.tr/api/v1/connector-configurations \
  -H "Authorization: Bearer $PAYVEN_TOKEN" \
  -H "Idempotency-Key: cfg-garanti-acme-prod" \
  -H "Content-Type: application/json" \
  -d '{
    "name":              "Garanti — Acme E-Ticaret Production",
    "connector_id":      "abc-...",
    "merchant_id":       "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "is_active":         true,
    "settings": {
      "store_id":      "1234567",
      "terminal_id":   "00012345",
      "store_key":     "***",
      "username":      "PROVAUT",
      "password":      "***",
      "endpoint_url":  "https://sanalposprov.garanti.com.tr/VPServlet",
      "client_id":     "0",
      "submerchant_country_code": "792"
    }
  }'
```

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| `name` | string | ✅ | Konfigürasyon adı (raporlama için) |
| `connector_id` | UUID | ✅ | Hangi konnektör tipini referans aldığı |
| `merchant_id` | UUID | ✅ | Hangi merchant adına çalışacak |
| `is_active` | bool | ❌ | Default `true`; pasif olarak oluşturulup test sonrası aktive edilebilir |
| `settings` | object | ✅ | Connector-spesifik credential dict — anahtarlar konnektöre göre değişir |

`settings` içerikleri her konnektör için farklıdır. Konsoldan **Konfigürasyon Ekle** sihirbazı ile alan listesini interaktif görebilir veya ilgili konnektör entegrasyon dokümanına bakabilirsiniz.

## Credential maskeleme

Hassas alanlar (`store_key`, `password`, vb.) yanıtlarda **maskelenmiş** halde döner. Açık değer yalnızca:

- Oluşturma yanıtında bir kez (sadece create eden için)
- Konsoldaki **Düzenle** akışında re-entry istediğinizde

görünür. Tekrar görmek için yeni bir konfigürasyon oluşturmanız veya konsoldan re-enter yapmanız gerekir.

## Aktif / pasif

```bash
# Pasife al
curl -X POST https://vpos.payven.com.tr/api/v1/connector-configurations/{id}/deactivate \
  -H "Authorization: Bearer $PAYVEN_TOKEN"

# Aktif et
curl -X POST https://vpos.payven.com.tr/api/v1/connector-configurations/{id}/activate \
  -H "Authorization: Bearer $PAYVEN_TOKEN"
```

Pasif konfigürasyonlar yönlendirme havuzundan çıkarılır. Test ortamında geçici devre dışı bırakma için faydalıdır.

## Sağlık endpoint'i

```bash
curl https://vpos.payven.com.tr/api/v1/connector-configurations/{id}/health \
  -H "Authorization: Bearer $PAYVEN_TOKEN"
```

Yanıt: anlık başarı oranı, son rolling pencere içindeki istek sayısı, ortalama yanıt süresi (ms), circuit breaker durumu (`closed` / `open` / `half_open`), son hata bilgisi.

## Yol haritası

- **Bağlantı testi endpoint'i** (`POST /{id}/test`) — credential'ların doğruluğunu canlı test
- **Schema endpoint'i** (`GET /schemas/{connector_code}`) — programatik konfigürasyon sihirbazları için zorunlu alan listesi

Şu anda bu özellikler konsoldan ulaşılabilir; public API'da yol haritasındadır.

## Hata yanıtları

| HTTP | `code` | Anlam |
|---|---|---|
| `404` | `connector_not_found` | Belirtilen `connector_id` bulunamadı |
| `404` | `merchant_not_found` | `merchant_id` bulunamadı |
| `422` | `validation_failed` | `settings` zorunlu alanları eksik |
| `409` | `duplicate_configuration_name` | Aynı isimde başka konfigürasyon var |

Hata zarfı RFC 9457 problem+json formatındadır.
