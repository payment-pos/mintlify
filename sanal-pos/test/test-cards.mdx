Test kart numaraları **bankanın test ortamı tarafından** belirlenir; her banka konnektörü kendi test kart matrisini sağlar. Payven sandbox'ında hangi kartların geçerli olduğu, kuruluşunuzun aktif konnektör konfigürasyonlarına bağlıdır.

## Konsoldan güncel matrisi alın

Sandbox konfigürasyonlarınıza göre derlenmiş, sürekli güncellenen test kartı listesi konsolda mevcuttur:

→ [Konsol → Test Kartları](https://dashboard.payven.com.tr/test-cards)

Bu ekran şunları gösterir:

- **Başarılı işlem kartları** — Visa / Mastercard / Troy / Amex
- **Banka tarafı ret senaryoları** — yetersiz bakiye, kart bloke, limit aşımı, geçersiz CVV
- **3DS test akışı** — challenge / frictionless tetikleyen kartlar, test SMS / şifre
- **Taksit testi** — kart programı (Bonus / Maximum / Axess / World) bazlı destekli taksit aralıkları
- **Geçerli son kullanma tarihi ve CVV** — her kart için bankanın kabul ettiği değerler

## Test akışı modları

Sandbox'ta iki farklı çalışma modu vardır — bkz. [Sandbox Ortamı](/sanal-pos/test/sandbox#banka-cagri-modu):

| Mod | Test kartı kaynağı |
|---|---|
| **Mock konnektör** | Payven tarafı sahte yanıt üretir. Numara biçimi (Luhn) doğruysa tipik senaryolar simüle edilebilir. |
| **Banka test ortamı konnektörü** | Bankanın resmi test kart listesi geçerlidir. Bu kartlar dışındaki numaraları banka reddeder. |

Hangi konfigürasyona ait olduğunuza göre yukarıdaki konsol ekranı doğru kartları otomatik filtreler.

## Üretim simülasyonu

Yüksek hacim testi için Payven **load test ortamı** sunar. Anlaşmanıza dahil olup olmadığını kontrol edin — değilse [destek ekibine](/resources/support) yazın.

## PCI uyumu

Sandbox test kartları **gerçek kart değildir**, dolayısıyla saklamak/loglamak teknik olarak ihlal değildir. Yine de production kodunuzun gerçek kartı işlemediğinden emin olmak için sandbox'ta da aynı disiplinle çalışın — kart numarasını veritabanına/loga yazmayın.
