---
title: "Logout"
description: "Aktif oturumu sonlandırma ve refresh token'ı geçersiz kılma."
---

Logout endpoint'i refresh token'ı geçersiz kılar — access token kalan süresince hâlâ kullanılabilir, ama yenileme yapılamaz. Tam logout için access token'ı da istemci tarafında silin.

## Endpoint

```http
POST /api/v1/auth/{slug}/logout
```

## İstek

```bash
curl -X POST https://identity.payven.com.tr/api/v1/auth/payven/logout \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "eyJhbGciOiJIUzI1NiI..."
  }'
```

| Alan | Tip | Zorunlu |
|---|---|---|
| `refresh_token` | string | ✅ |

<Note>
Bu endpoint anonim erişilebilir — `Authorization: Bearer ...` header'ı gerekmez. Refresh token kendi kendine kimlik bilgisidir.
</Note>

## Yanıt

`204 No Content` — gövde yok.

## Hata yanıtları

| HTTP | `code` | Anlam |
|---|---|---|
| `404` | `tenant_not_found` | Slug geçersiz |
| `401` | `invalid_refresh_token` | Refresh token geçersiz / zaten geçersizleştirilmiş |

## En iyi uygulamalar

<Check>**Logout her zaman çağrılsın** — sadece istemci tarafında token silmek yetersiz; çalınmış bir refresh token hâlâ kullanılabilir.</Check>
<Check>İstemci tarafında tüm token'ları temizleyin (cookie, in-memory state, localStorage).</Check>
<Check>Cross-tab senaryoda diğer sekmelere `storage` event'i ile haber verin → her sekme kendi state'ini sıfırlasın.</Check>
<Check>Logout başarısız olsa bile (network kesintisi vb.) istemci tarafında token'ları silin — refresh denendiğinde sunucu reddedecektir.</Check>
