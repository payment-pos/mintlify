---
title: "Login (Portal Kullanıcısı)"
description: "İnsan kullanıcılar için kullanıcı adı / parola ile JWT alma."
---

Identity servisi üzerinden **konsol kullanıcısı** (insan) oturumu başlatır. Sunucu-sunucu entegrasyonları için bu yerine [Token (Client Credentials)](/documentation/concepts/authentication) endpoint'ini kullanın.

## Endpoint

```http
POST /api/v1/auth/{slug}/login
```

| Path parametresi | Açıklama |
|---|---|
| `slug` | Kuruluşunuzun slug'ı (onboarding sırasında verilen). |

<Note>
Bu endpoint **rate limited**: IP başına dakikada **10** istek. Aşımda `429 rate_limit_exceeded`.
</Note>

## İstek

```bash
curl -X POST https://identity.payven.com.tr/api/v1/auth/payven/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "user@example.com",
    "password": "**********"
  }'
```

| Alan | Tip | Zorunlu |
|---|---|---|
| `username` | string | ✅ |
| `password` | string | ✅ |

## Yanıt

`200 OK` — `AuthTokenResult`:

```json
{
  "access_token":        "eyJhbGciOiJSUzI1NiI...",
  "refresh_token":       "eyJhbGciOiJIUzI1NiI...",
  "expires_in":          300,
  "refresh_expires_in":  1800,
  "token_type":          "Bearer",
  "scope":               "openid profile email"
}
```

| Alan | Açıklama |
|---|---|
| `access_token` | Sonraki API çağrılarında `Authorization: Bearer ...` header'ı ile gönderin |
| `refresh_token` | Access token süresi dolduğunda yenilemek için |
| `expires_in` | Access token saniye cinsinden geçerlilik (varsayılan **5 dakika**) |
| `refresh_expires_in` | Refresh token idle timeout (varsayılan 30 dakika; offline scope ile 30 gün) |
| `token_type` | Daima `Bearer` |
| `scope` | Tahsis edilen OIDC scope'lar |

## Token kullanımı

```http
GET /api/v1/me
Authorization: Bearer eyJhbGciOiJSUzI1NiI...
```

## Hata yanıtları

Hata zarfı [RFC 9457 problem+json](/documentation/concepts/errors).

| HTTP | `code` | Anlam |
|---|---|---|
| `401` | `invalid_credentials` | Kullanıcı adı veya parola hatalı |
| `403` | `user_disabled` | Hesap devre dışı |
| `403` | `user_pending_approval` | Onay bekliyor |
| `404` | `tenant_not_found` | Slug geçersiz |
| `429` | `rate_limit_exceeded` | Çok fazla başarısız deneme — `Retry-After` header'ına uyun |

## Güvenlik

<Check>Login isteklerini **sadece HTTPS** üzerinden atın.</Check>
<Check>Tarayıcıda `access_token`'ı in-memory veya `httpOnly` cookie'de tutun, `localStorage`'a yazmayın.</Check>
<Check>`refresh_token`'ı sadece sunucu tarafında saklayın — tarayıcıya açmayın.</Check>
<Check>Brute-force koruması: ardışık başarısız giriş denemelerinde rate limit ve hesap kilidi devreye girer.</Check>
