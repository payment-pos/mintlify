Payven, Türkiye'nin finansal teknoloji altyapısıdır. **14 banka** ile entegre tek bir API üzerinden sanal POS, para transferi, fraud yönetimi ve mutabakat modüllerini kullanırsınız. Lisanslı ödeme kuruluşları ve büyük platformlar; banka entegrasyonlarının teknik karmaşıklığını Payven'e bırakır, kendi ürünlerine odaklanır.

Fatura ödeme ve açık bankacılık modülleri yol haritasındadır — bkz. [Yol haritası](#yol-haritasi).

## API'a ilk bakış

Authorization header'ı + JSON body — beklediğiniz gibi:

```bash
curl https://vpos.payven.com.tr/api/v1/payments \
  -H "Authorization: Bearer $PAYVEN_TOKEN" \
  -H "Idempotency-Key: order-1001" \
  -H "Content-Type: application/json" \
  -d '{
    "external_id": "ORDER-1001",
    "amount":      { "amount": 15000, "currency": "TRY" },
    "installment": 1,
    "card": {
      "holder_name":  "Test Kullanici",
      "number":       "4546711234567894",
      "expire_month": "12",
      "expire_year":  "2030",
      "cvv":          "000"
    }
  }'
```

Token alımı, 3D Secure akışı ve webhook doğrulaması için: [Hızlı Başlangıç](/documentation/quickstart).

## Entegrasyon adımları

<Steps>
  <Step title="Kuruluş hesabınızı açın">
    [Konsola](https://dashboard.payven.com.tr) erişim için satış ekibimizle iletişime geçin. Onboarding 1-2 iş günü sürer.
  </Step>
  <Step title="API anahtarınızı oluşturun">
    Konsoldan **client_id** ve **client_secret** çifti üretin. Secret yalnızca oluşturma anında gösterilir, güvenli bir vault'ta saklayın.
  </Step>
  <Step title="Sandbox'ta token alın">
    Identity'den OAuth 2.0 client_credentials akışıyla access token alın. Kod örnekleri (Node, Python, C#, Go, PHP) auto-refresh dahil hazır.
  </Step>
  <Step title="İlk ödemeyi gerçekleştirin">
    [Hızlı Başlangıç](/documentation/quickstart) — curl ile çalışan örneklerle 3DS akışını test edin.
  </Step>
  <Step title="Webhook ile asenkron yakalayın">
    HTTPS endpoint'inizi kaydedin, [HMAC-SHA256 imzasını](/sanal-pos/webhooks/signature) doğrulayın.
  </Step>
  <Step title="Production'a geçin">
    [Go-live kontrol listesi](/documentation/security/go-live-checklist) — IP whitelist, secret yönetimi, rate limit, izleme.
  </Step>
</Steps>

## Payven'i farklı kılan ne?

<CardGroup cols={2}>
  <Card title="Tek Kimlik, Tüm Ürünler" icon="layer-group">
    Tek bir access token planınızdaki bütün modüllere geçer. Sanal POS, para transferi, fraud — hepsi aynı bearer token ile çağrılır, ayrı kimlik akışı yok.
  </Card>
  <Card title="Akıllı Yönlendirme + Smart Retry" icon="route">

  <Card title="Standartlara uyumlu API" icon="check-double">
    HTTP semantiğine sıkı uyum: doğru status kodları, RFC 9457 `application/problem+json` hata zarfı, `Idempotency-Key` desteği, sayfalı listelemeler, tutarlı snake_case alan adları.
  </Card>
  <Card title="Operasyonel Şeffaflık" icon="chart-line">
    Konsolda her API çağrısının korelasyon zinciri, banka yanıt detayı, retry geçmişi ve webhook teslim logu — debug saatleri değil dakikaları alır.
  </Card>
</CardGroup>

## Ne zaman Payven'i tercih etmelisiniz?

Payven doğrudan son tüketiciye hizmet vermez. **Lisanslı ödeme kuruluşları, bankalar ve büyük platformlar** için altyapı sağlar.

| Profil | Uygun mu? |
|---|---|
| TCMB lisanslı ödeme veya elektronik para kuruluşu — alt bayilerinize ödeme hizmeti sunuyorsunuz | ✅ Birincil hedef |
| Banka — sanal POS gateway altyapınızı modernize etmek istiyorsunuz | ✅ Uygun |
| Büyük e-ticaret / marketplace — birden fazla bankayı tek API'den yönetmek istiyorsunuz | ✅ Uygun |
| Yüksek hacimli fintech / scale-up — orchestration katmanı arıyorsunuz | ✅ Uygun |
| Bireysel geliştirici / küçük e-ticaret sitesi | ❌ Doğrudan PSP'lerle çalışmanız önerilir |

## Yol haritası [#yol-haritasi]

Aşağıdaki modüller geliştirme aşamasındadır; canlı API'ye dahil değildir. Erken erişim için satış ekibimizle iletişime geçin.

- **Fatura Ödeme** — Kurum, GSM ve fatura tahsilat entegrasyonları.
- **Açık Bankacılık** — Hesap bilgisi (AIS) ve ödeme başlatma (PIS) servisleri.

Yenilikler ve sürüm notları için: [Changelog](/resources/changelog).
